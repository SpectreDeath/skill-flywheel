# Plan: Migrate Multi-Surface Core to em-cubed Repository

## Objective
Extract multi-surface (multi-logic) core infrastructure from Skill Flywheel into standalone `em-cubed` repo at `D:\GitHub\projects\em-cubed`, keeping 878 domain skills and content in original repo.

## Key Decisions
- **em-cubed**: Standalone Python package for multi-surface skill management (indexing, search, surface execution)
- **Skill Flywheel**: Content repo with domain skills, using em-cubed as dependency
- **Registry JSON**: Build artifact in Skill Flywheel, generated by em-cubed indexer

## Phase 1: Initialize em-cubed Structure
Create base structure at `D:\GitHub\projects\em-cubed`:
```
em-cubed/
```

## Phase 2: Extract & Refactor Core
### 2.1 Refactor Indexer (reindex_skills.py → indexer.py)
- Remove hardcoded paths (`WORKSPACE_ROOT`, `SKILLS_DIR`, `REGISTRY_FILE`)
- Make configurable:
  - `skills_dir: Path` (required)
  - `registry_output: Path` (required)
- Preserve logic: fenced block extraction, Prolog/Hy tag extraction, YAML frontmatter parsing
- Move tag extraction functions to respective surface modules

### 2.2 Refactor Search (registry_search.py → search.py)
- Remove hardcoded `REGISTRY_FILE` path
- Make configurable: `registry_path: Path` (required)
- Preserve scoring logic for surfaces/tags

### 2.3 Move Documentation & Examples
- Move `docs/MULTI_SURFACE.md` → `em-cubed/docs/`
- Move `domains/COGNITIVE_SKILLS/multi-surface-problem-solver/` → `em-cubed/examples/`
