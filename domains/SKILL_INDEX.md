## NEW: Folder Organization Rules

### Core Principles
- **NEVER DELETE SKILLS**: Use folder organization instead of deletion
- **29 SKILL/ MAX**: Production core limited to 29 skills for maintainability
- **DOMAIN/ FOR SPECIALIZATION**: Domain-specific skills in dedicated folders
- **EXPERIMENTAL/ FOR INNOVATION**: Safe space for chaos output and experiments
- **ARCHIVED/ FOR HISTORY**: Deprecated skills preserved for reference

### Organization Workflow
1. **New skills** → Evaluate for SKILL/ (core) vs DOMAIN/ (specialized)
2. **Domain skills** → Route to appropriate DOMAIN/[domain_name]/ folder
3. **Chaos output** → Store in EXPERIMENTAL/ for review and refinement
4. **Deprecated skills** → Move to ARCHIVED/ (never delete)
5. **Core skills** → Maintain in SKILL/ (max 29 total)

### Domain Detection Rules
- **Game Dev**: Unity, performance profiling, multiplayer architecture
- **Web3**: Blockchain, smart contracts, DeFi, cryptocurrency
- **DevOps**: CI/CD, containers, infrastructure, monitoring
- **ML/AI**: Machine learning, deep learning, computer vision, MLOps
- **Frontend**: React, UI/UX, state management, build tooling
- **Specification Engineering**: PRD, technical specs, API design, ADRs, test plans
