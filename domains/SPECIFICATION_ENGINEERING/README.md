# Specification Engineering

Domain for skills related to specification-driven development, including specification validation, traceability, testing, lifecycle management, and evolution.

## Skills

| Skill | Description |
|-------|-------------|
| [specification-validation-framework](specification-validation-framework/) | Automated testing and validation for specifications |
| [specification-traceability](specification-traceability/) | Requirement-to-implementation traceability |
| [executable-spec-harness](executable-spec-harness/) | Generate tests from specification documents |
| [spec-regression-monitoring](spec-regression-monitoring/) | Track spec drift and detect breaking changes |
| [spec-evolution-engine](spec-evolution-engine/) | Manage spec versioning with backward compatibility |
| [specification-lifecycle-management](specification-lifecycle-management/) | Automated spec review cycles and expiration |
| [spec-contract-authoring](spec-contract-authoring/) | Author executable specifications (OpenAPI, JSON Schema) |
| [spec-to-task-decomposition](spec-to-task-decomposition/) | Break specs into actionable implementation tasks |
| [spec-guardrail-enforcement](spec-guardrail-enforcement/) | Enforce spec compliance during implementation |

## Usage

Load these skills when working with specifications in software development projects:
- Validating specification quality before implementation
- Generating automated tests from specs
- Tracking requirements through implementation
- Managing API versioning and backward compatibility
- Maintaining specifications over time
- Breaking down specs into tasks
- Enforcing spec compliance in code
